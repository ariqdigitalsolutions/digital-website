Help & Updates

System changes, new functionality, and quick instructions for cashiers and supervisors.

New Features

  • Split Payment: USD and ZWG sales can now be paid using cash and POS (Swipe / EcoCash) together on the same receipt.
  • Automatic Receipt Printing: completed sales now send a receipt print command with sale details and a fiscalisation QR-code area at the bottom.
  • Transfer Receipt Printing: transferred transactions print the barcode receipt in the background instead of opening a separate print screen.
  • Cashup Signatures: cashup printouts now include cashier and supervisor signature spaces.

System Changes

  • Cashier users no longer see the dashboard panel; they start directly from Record Sale.
  • The Record Sale page layout has been reverted to the previous setup: Add Product at the top, with Current Receipt and Payment below.
  • ZAR remains cash only; USD and ZWG allow split payment by cash and POS (Swipe / EcoCash).
  • Transaction success now displays a simple success message and then prints the receipt.

How to Use

  1. Record sale: use the Add Product panel at the top, select product, enter quantity, click Add / Enter, then repeat for all products.
  2. Split payment: choose USD or ZWG, enter the cash amount and POS (Swipe / EcoCash) amount, then click Record Sale. The combined amount must cover the receipt total.
  3. ZAR sale: choose ZAR and enter cash only. POS / EcoCash is disabled for ZAR.
  4. Transfer transaction: click Transfer Transaction, choose Transfer Transaction, and the barcode slip prints automatically. The receiving cashier scans or types the code under Receive Transaction.
  5. Cashup: select currency, compare system totals against counted money, record cashup, then another supervisor concludes and signs the printout.

Important Notices

  • Normal browsers may still show the system print dialog unless the till computer is configured for kiosk/silent printing.
  • The fiscalisation QR area is ready for ZIMRA integration. Once the real ZIMRA response is connected, the official fiscal QR data should be passed into the receipt.
  • A supervisor cannot conclude their own cashup; another supervisor must conclude and sign it.
  • Always run the latest Supabase SQL patch before uploading the new POS files.
