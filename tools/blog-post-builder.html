Local publishing tool

Prepare a new blog article

This private, unlinked tool creates a safe article object that you can paste into the blog data file. It does not request or store a GitHub password or token.

How publishing works

  1. Upload the article image to the project.
  2. Complete this form and copy the generated object.
  3. Paste it into assets/js/blog-data.js.
  4. Commit and publish through GitHub Pages.

Article details

Create the post object

Nothing is uploaded automatically. The output remains in your browser until you copy it.

Use a project-relative path. Upload the image first.
Use ## Heading on its own line for section headings. Blank lines separate paragraphs.

Generated output

Paste this object inside the window.ARIQ_BLOG_POSTS = [ ... ] array and add a comma between posts.

Security note

Never paste a GitHub token, password, API key or private credential into a public website file. Use the GitHub website or desktop application to upload and commit changes.
